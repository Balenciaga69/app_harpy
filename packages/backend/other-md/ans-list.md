戰鬥對戰改成:
(以下比較像是 API層與前端層任務)
後端不在發送 snapshot list 這樣封包太多太誇張
當 Client 上傳的結果與 Backend 模擬的結果不同時伺服器會直接拒絕該次戰鬥結算。玩家畫面上可能顯示「贏了」但會彈出「伺服器連接錯誤」或「數據異常」，導致戰鬥無效。會回到戰鬥前，且不認可該次對戰。
理論上前端也是不到一秒就跑完運算結果，並與後端比對 但如果有異常也是到演戲完畢取得正確結果才執行後續。

而是前端運行一次 後端也運行一次。前端返回對戰結果 如果與前端的勝利方以及剩餘生命都一樣，代表沒作弊。
前端負責「演戲」給玩家看。後端審判演戲結果與他自己模擬的是否一樣，因為我們種子碼一樣。

Aliment 基礎構想:

- 中毒: 1layer = -1hp 無上限
  - 1000 tick 減少 10%，最少 -10
- 冰緩: 1layer = -1% AttackSpeed 上限 64Layer
  - 1000 tick 減少 1， attack 也 -1
- 充能: 1layer = +1% AttackSpeed 上限 64Layer
  - 1000 tick 減少 1， attack 也 -1
- 致盲: 1layer = -1% 命中 上限 32Layer
  - 1000 tick 減少 1， attack 也 -1
- 嗜血: 1layer = +1% 攻擊傷害吸血,+1%暴擊率,+2%暴擊加成; 8layer max
  - 1000 tick 減少 1， attack 也 -1
- 每個單位的 Tick 計時器根據其生成時間有微小的 Offset
