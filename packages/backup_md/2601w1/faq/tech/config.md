# Config 通用問答

## Q: Config 是什麼？

A: Config 是遊戲所有靜態資料的總稱，例如職業模板、物品模板、敵人模板、詞綴模板、大絕招模板等。Config 只保存「不會隨玩家行為改變」的內容，這些資料由設計師維護，通常以表格、JSON 等格式儲存。

## Q: Config 在遊戲中的角色是什麼？

A: Config 是遊戲世界的「規則書」與「內容字典」，決定了所有可用內容、生成規則、數值範圍與條件。遊戲每次啟動時，會將所有 Config 載入記憶體，供遊戲邏輯查詢使用。

## Q: 設計師如何使用 Config？

A:

- Config 是設計遊戲內容的唯一入口
- 只需維護表格與靜態資料，不需理解程式碼
- 所有遊戲內容都能透過 Config 快速調整與擴充
- 修改模板內容後，由 Loader 載入並組裝進遊戲

## Q: 工程師如何實現 Config 系統？

A: Config 系統由四部分組成：

- **Config Loader**：從外部來源（JSON、資料庫、表格）載入靜態資料並轉換格式
- **Config Store**：將載入的資料存放於記憶體，提供高效查詢介面（如 getItem、getEnemy）
- **Config Assembler**：統一組裝與初始化所有靜態資料，處理資料間的關聯與驗證
- **Config DTO**：定義 Loader 載入時的資料格式

## Q: Config 系統有什麼優點？

A:

- 靜態資料與邏輯分離，提升維護性
- 靜態資料變更不會影響程式碼穩定性，方便測試與部署
- 支援熱更新、快速調整內容
- 所有靜態資料都經過統一流程載入與查詢，避免資料不一致

## Q: 什麼時候應該用 Config？

A: 任何需要「不會隨玩家行為改變」的資料都應該從 Config 系統取得：

- 查詢職業、物品、敵人、詞綴、大絕招等靜態資料
- 生成遊戲實例時必須先查詢對應模板
- 當設計師要調整遊戲內容時，只需修改 Config，不需動到遊戲邏輯
