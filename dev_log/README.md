# v0.3 摘要

_2025/11/30_

今天是禮拜六，我終於完成了基於 Tick 機制的戰鬥系統初步原型。為了能支援高度複雜且非線性的戰鬥邏輯，其複雜度超乎想像。我想要達成類似《流亡黯道》中「攻擊物理傷害轉換為火焰」、「承受冰傷害視為毒傷害」，或「攻擊暴擊時觸算兩次」等特殊的運算邏輯，我在架構設計與數據模型建構上投入了大量的時間與精力。

在開發過程中，GitHub Copilot 提供了極大的助力。我主要負責核心架構的構思、數據模型圖的繪製、以及設計模式與架構權衡 (Trade-off) 的決策；而 Copilot 則高效地生成了近 99% 的代碼，我僅需進行最後的微調與整合。

然而，在開發進行到一半時，我重新評估了遊戲機制。我接觸並研究了《聯盟戰棋》、《選技大亂鬥》、《背包大亂鬥》等遊戲的設計，發現這些機制的核心理念與輕量化方向可能更符合我當前專案的遊戲設計需求，這促使我做出了一個重大決定：從遊戲設計層面重新調整戰鬥系統的核心機制。

接下來的調整方向包括：新增「大招系統」，並移除或大幅簡化複雜的元素抗性與多重傷害轉換機制。值得慶幸的是，由於前期採用了乾淨且模組化的架構設計，變動帶來的連鎖影響得以控制在最小範圍。
